# Prompts

Два активных промпта, запускаются последовательно для каждого джоба.

## Активные

**filter.txt** — Этап 1, дешёвая модель (gemini-2.5-flash-lite)
Один вопрос: относится ли джоб к нишам? (автоматизация, outreach, CRM, Python, scraping, веб)
Если нет — джоб выбрасывается, этап 2 не запускается.
Возвращает: `{ relevant, niche, reason }`

**score.txt** — Этап 2, умная модель (gemini-2.5-flash-lite)
Детальный скоринг только для релевантных джобов.
Учитывает: новизну клиента, возможность демки, конкуренцию, шансы с низким рейтингом.
Возвращает: `{ overall_score, quick_demo_possible, newcomer_friendly, reasoning, rejection_reasons, tags }`

## Как менять

Модели и имена промптов задаются в `config.js`:
- `filterModel` / `filterPrompt` — этап 1
- `scoreModel` / `scorePrompt` — этап 2

Чтобы поменять ниши — редактируй `filter.txt`.
Чтобы поменять критерии скоринга — редактируй `score.txt`.
Схема БД при этом не трогается.

## Архив

`archive/` — старые промпты (single-stage подход). Не используются.
